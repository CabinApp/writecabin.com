Roadmap

From quiet idea to closed alpha.

The path begins with product definition, then moves through research, design, prototype, core editor, project structure and alpha testing.

Completed

Identity and philosophy

Name, domain, GitHub organization, logo, typography, color palette and the first product philosophy are in place.

In progress

Personas and research

Understand first-time novelists, worldbuilders, screenwriters, researchers and students before choosing the first feature set.

Next

MVP selection

Turn the large feature inventory into a restrained first version, with everything else placed deliberately into later work.

Planned

UX and file model

Sketch layouts, panels, resizing, docking, views and the project folder structure before implementation begins.

Planned

Design system

Build spacing, typography, icons, menus, sidebars, modals and themes so the interface feels calm at every scale.

Planned

Prototype

Create a pure interface prototype to test resizing, docking, layouts and motion before connecting persistence.

Planned

Core editor

Implement projects, documents, the editor, autosave, themes and tabs with reliability as the baseline.

Planned

Project system

Add folders, chapters, scenes, drag and drop ordering, metadata, workspace presets, search and exports.

Alpha

Closed alpha

Recruit a small group of writers, observe without over-explaining, take notes and repair the rough edges.
