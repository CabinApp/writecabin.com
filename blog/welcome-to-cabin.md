# Welcome to Cabin

Cabin is writing software built around a simple belief: the tool should grow at the same pace as the writer's project.

## Why begin with calm?

Many writing apps begin by asking the writer to manage a system. Cabin begins with the page, then lets structure arrive only when the work can use it.

The early principles are plain:

- Keep the manuscript central.
- Make ownership visible.
- Let planning tools stay quiet until needed.
- Treat AI as an editor, never a co-author.

The order matters:

1. Understand the writer's actual working rhythm.
2. Design the smallest useful writing room.
3. Add structure without turning the app into a dashboard.
4. Test the alpha with real long-form projects.

> Cabin should feel like a quiet room prepared for thought, not a dashboard pretending to be progress.

## Small table

| Principle | Meaning |
| --- | --- |
| Local-first | The writer owns the folder. |
