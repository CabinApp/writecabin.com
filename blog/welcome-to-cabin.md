# Welcome to Cabin

Cabin is writing software built around a simple belief: the tool should grow at the same pace as the writer's project.

## What the blog supports

This mock article exists so you can see the Markdown renderer in motion.

- Headings
- Paragraphs and emphasis
- Ordered and unordered lists
- Blockquotes
- Tables
- Links
- Inline code
- Syntax-highlighted code blocks

1. Add a Markdown file to the `blog` folder.
2. Give it frontmatter with `title`, `date` and `excerpt`.
3. Add the title, date, excerpt, path and slug to `blog/index.json`.
4. Push to GitHub Pages.

> Cabin should feel like a quiet room prepared for thought, not a dashboard pretending to be progress.

## Small table

| Principle | Meaning |
| --- | --- |
| Local-first | The writer owns the folder. |
| Progressive complexity | Structure appears when useful. |
| AI as editor | The writer keeps the voice. |

## Code block

```js
const cabin = {
  startsAs: "one blank file",
  growsWhen: "the writer asks",
  aiRole: "editor, not co-author"
};

console.log(`${cabin.startsAs}; ${cabin.growsWhen}.`);
```
